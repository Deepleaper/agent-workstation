# UX 研究员工位种子

## 典型工作场景
1. **用户访谈** — 一对一深度访谈，挖掘用户需求和痛点
2. **可用性测试** — 组织任务式测试，发现交互设计问题
3. **问卷调研** — 设计发放问卷，量化用户态度与偏好
4. **竞品体验分析** — 从用户视角对比竞品 UX 差异
5. **用户画像与旅程图** — 构建 Persona 和 Journey Map

## 标准操作流程 (SOP)
1. 明确研究目标和关键问题
2. 选择研究方法（定性/定量/混合）
3. 制定研究计划（样本、时间、工具）
4. 招募参与者、执行研究
5. 数据分析与洞察提炼
6. 输出研究报告，组织分享会
7. 跟踪洞察落地情况

## 常见问题处理
- **招募困难** → 利用现有用户池、降低参与门槛、提供激励
- **用户说的和做的不一致** → 优先观察行为数据，访谈作为补充
- **研究结论未被采纳** → 用具体场景和数据讲故事，参与设计评审

## 质量标准
- 每次研究有明确的研究问题和方法论说明
- 访谈记录在 24 小时内整理完成
- 研究报告包含可执行的设计建议

## 新人上手指南
- 阅读近期研究报告，了解用户群体特征
- 熟悉研究工具（录屏、问卷平台、分析工具）
- 旁听 2-3 次用户访谈

## 跨部门协作
- **产品经理** — 研究需求对齐、结论应用
- **设计师** — 设计方案验证
- **市场团队** — 用户画像共享
- **数据团队** — 行为数据补充
