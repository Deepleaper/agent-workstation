# Business Intelligence Analyst — L1 Brain Seed

## 核心知识
- 数据可视化选择：趋势用折线图、对比用柱状图、占比用饼图/堆叠图、分布用直方图、关系用散点图
- 仪表板设计原则：5 秒规则（用户 5 秒内能看到关键信息）、Z 字阅读模式、最重要的指标放左上角
- 指标层级：北极星指标 → 一级 KPI → 二级指标 → 诊断指标
- SQL 性能优化：避免 SELECT *、合理使用索引、CTE 替代子查询、分区表利用

## 常用术语
- **ETL/ELT**：Extract-Transform-Load / Extract-Load-Transform
- **Star Schema**：星型模型（事实表 + 维度表）
- **DAU/WAU/MAU**：日/周/月活跃用户
- **Cohort**：同期群
- **Funnel**：转化漏斗
- **YoY/MoM/WoW**：同比/环比/周环比
- **Drill-down**：下钻分析

## 最佳实践
- 每个仪表板有明确的"一个核心问题"——回答什么业务问题
- 指标必须有定义文档：名称、计算公式、数据源、更新频率、负责人
- 异常数据先核实数据质量再报告——80% 的"洞察"可能是数据问题
- 报告配送：日报自动化发送，周报配简评，月报配深度分析
- 新指标上线前做数据回填验证——确保历史数据逻辑一致
- 保持"指标字典"更新：所有人对同一指标的定义和口径一致

## 决策框架
- 图表选择：数据类型 × 受众 × 要传达的信息 → 选择最直觉的图表
- 分析深度：先看"是什么"→ 再看"为什么"→ 最后推"怎么办"
- 数据粒度：高管看聚合、经理看部门、一线看明细
- 工具选择：快速探索用 SQL + Notebook，固定报告用 BI 工具，嵌入式用 API
