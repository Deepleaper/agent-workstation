# 数据分析方法论框架

## 一、分析流程（CRISP-DM）

### 1. 业务理解
- 明确业务目标和成功标准
- 确定分析范围和约束条件
- 制定分析计划和时间表
- **关键问题**：我们要解决什么问题？决策者需要什么？

### 2. 数据理解
- 数据源识别和获取
- 数据质量评估
- 初步探索性分析（EDA）
- **核心动作**：
  - 数据字典整理
  - 缺失值/异常值统计
  - 基本分布和相关性查看

### 3. 数据准备
- 数据清洗（缺失值、异常值、重复值处理）
- 数据转换（标准化、编码、特征工程）
- 数据集构建（训练集/测试集划分）

### 4. 建模/分析
- 选择分析方法
- 构建模型或分析框架
- 参数调优和验证

### 5. 评估
- 结果是否回答了业务问题？
- 模型性能是否达标？
- 是否有未考虑的因素？

### 6. 部署/输出
- 分析报告撰写
- 可视化Dashboard
- 自动化和监控方案

## 二、常用分析框架

### 2.1 描述性分析
- **用途**：发生了什么？
- **方法**：统计汇总、趋势分析、分布分析
- **工具**：SQL聚合、Excel透视表、BI工具

### 2.2 诊断性分析
- **用途**：为什么发生？
- **方法**：下钻分析、对比分析、相关性分析
- **常用框架**：
  - 漏斗分析（转化率逐层拆解）
  - 同期群分析（Cohort Analysis）
  - 归因分析（Attribution）

### 2.3 预测性分析
- **用途**：将会发生什么？
- **方法**：时间序列预测、回归分析、分类模型
- **适用场景**：销量预测、用户流失预测、风险评估

### 2.4 规范性分析
- **用途**：应该怎么做？
- **方法**：A/B测试、优化模型、决策树
- **适用场景**：定价策略、推荐系统、资源分配

## 三、核心分析指标体系

### 用户指标
| 指标 | 公式 | 说明 |
|------|------|------|
| DAU/MAU | 日/月活跃用户数 | 产品活跃度 |
| 留存率 | 第N天回访用户/新增用户 | 用户粘性 |
| LTV | 用户生命周期总价值 | 用户价值 |
| CAC | 获客总成本/新增用户数 | 获客效率 |
| ARPU | 总收入/活跃用户数 | 用户贡献 |

### 业务指标
| 指标 | 公式 | 说明 |
|------|------|------|
| 转化率 | 转化人数/访问人数 | 漏斗效率 |
| GMV | 订单总金额 | 交易规模 |
| 客单价 | GMV/订单数 | 消费水平 |
| 复购率 | 重复购买用户/总购买用户 | 忠诚度 |

## 四、数据可视化原则

1. **一图一义**：每张图表只表达一个核心观点
2. **选对图表**：
   - 趋势 → 折线图
   - 对比 → 柱状图
   - 占比 → 饼图/环形图
   - 分布 → 直方图/箱线图
   - 关系 → 散点图
3. **去除噪音**：删除不必要的网格线、标签、装饰
4. **突出重点**：用颜色或标注引导注意力
5. **讲故事**：图表要有标题、注释和结论
