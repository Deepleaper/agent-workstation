# Report Generator — L1 Brain Seed

## 核心知识
- 好报告的四要素：准确（数据对）、及时（按时交）、清晰（看得懂）、可操作（能指导决策）
- 报告受众分层：C-level 看趋势和决策点，VP 看部门表现，经理看团队明细
- 数据可视化原则：数据墨水比最大化（Edward Tufte）——去掉不传递信息的元素
- 数字对比三维度：同比（YoY）、环比（MoM/WoW）、预算达成率

## 常用术语
- **MTD/QTD/YTD**：月/季/年累计至今
- **Variance**：差异分析
- **Actuals vs Budget**：实际 vs 预算
- **Run Rate**：年化运行率
- **Waterfall Chart**：瀑布图（展示增减分解）
- **Pivot Table**：数据透视表
- **Drill-through**：穿透查询

## 最佳实践
- 报告第一页永远是执行摘要：3-5 个关键发现 + 建议行动
- 数字格式统一：千分位、小数位、百分比、货币符号——全报告一致
- 颜色编码：绿色=达标/正增长，红色=未达标/负增长，黄色=预警
- 定期报告在固定时间发送——培养读者期望和阅读习惯
- 每季度做一次"报告清理"：停掉没人看的报告，合并重复报告
- 数据来源和计算说明放在附录，正文保持简洁

## 决策框架
- 报告格式选择：决策型用 PPT/PDF，分析型用 Excel/BI，监控型用 Dashboard
- 自动化判断：每周重复 + 格式固定 = 必须自动化；临时性 + 格式灵活 = 手动
- 详略取舍：高管报告 ≤ 5 页，部门报告 ≤ 15 页，详细分析作为附件
- 数据冲突处理：标注两个来源的数据差异，说明推荐使用哪个以及原因
