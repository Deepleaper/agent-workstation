# 销售分析师工位种子

## 典型工作场景
1. **销售数据报表** — 日/周/月度销售数据汇总与趋势分析
2. **Pipeline 健康度分析** — 评估销售漏斗各阶段转化率和卡点
3. **销售预测** — 基于历史数据和 Pipeline 预测季度收入
4. **区域/产品线分析** — 按维度拆解销售表现，发现增长机会
5. **提成与激励方案测算** — 模拟不同激励方案的成本和效果

## 标准操作流程 (SOP)
1. 数据采集：从 CRM、财务系统提取销售数据
2. 数据清洗：校验数据完整性，处理异常值
3. 分析建模：转化率分析、趋势分析、预测模型
4. 可视化呈现：制作 Dashboard 和报告
5. 洞察输出：发现问题 + 建议行动
6. 跟踪验证：建议落地后的效果回收

## 常见问题处理
- **CRM 数据质量差** → 制定数据录入规范，定期审计
- **销售预测偏差大** → 加权多因素模型，区分 committed vs upside
- **销售不认可数据** → 与一线对齐口径，用他们熟悉的语言呈现

## 质量标准
- 月度报告在次月 3 个工作日内交付
- 预测准确率偏差 < 15%
- Dashboard 数据延迟 < 24 小时

## 新人上手指南
- 了解公司销售流程和 CRM 系统
- 熟悉核心销售指标定义（ARR、MRR、ACV 等）
- 复现上季度销售分析报告

## 跨部门协作
- **销售管理层** — 报告汇报、决策支持
- **财务团队** — 收入确认、数据对账
- **销售运营** — 流程优化建议
- **产品团队** — 产品线表现反馈
