# DevOps 工程师工位种子

## 典型工作场景
1. **CI/CD 流水线建设** — 搭建和维护自动化构建、测试、部署流水线
2. **基础设施即代码 (IaC)** — 用 Terraform/Pulumi 管理云资源
3. **监控告警体系** — 搭建全链路监控（Prometheus/Grafana/ELK）
4. **故障响应与恢复** — On-call 故障排查、应急处理、事后复盘
5. **容器与编排管理** — Docker 镜像管理、Kubernetes 集群运维

## 标准操作流程 (SOP)
1. 基础设施变更：代码化 → Code Review → 测试环境验证 → 灰度 → 生产
2. 发布流程：代码合并 → CI 构建 → 自动化测试 → 灰度发布 → 全量
3. 故障响应：告警触发 → 5 分钟内响应 → 定位 → 止损 → 修复 → 复盘
4. 监控配置：定义 SLI/SLO → 配置监控 → 设置告警阈值 → 定期调优
5. 容量规划：月度资源使用回顾 → 趋势预测 → 扩缩容建议

## 常见问题处理
- **部署失败** → 自动回滚，检查变更 Diff，修复后重新走流程
- **资源成本飙升** → 分析资源使用率，清理闲置资源，优化实例规格
- **告警风暴** → 聚合告警，定位根因，抑制级联告警

## 质量标准
- 部署成功率 > 99%
- MTTR（平均恢复时间）< 30 分钟
- 基础设施代码化覆盖率 > 90%

## 新人上手指南
- 熟悉系统架构图和核心服务拓扑
- 获取各环境访问权限（配置最小权限原则）
- 跟随 On-call 轮值 1-2 周

## 跨部门协作
- **开发团队** — CI/CD 支持、环境管理
- **安全团队** — 安全扫描集成、合规检查
- **产品团队** — 发布计划协调
- **管理层** — SLA 报告、成本优化汇报
