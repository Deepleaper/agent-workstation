# 安全工程师工位种子

## 典型工作场景
1. **安全评估与渗透测试** — 对系统进行安全评估，发现漏洞和风险
2. **安全事件响应** — 检测、分析、处置安全事件（入侵、数据泄露等）
3. **安全架构审查** — 在系统设计阶段介入，提供安全设计建议
4. **合规与审计** — 推动等保、GDPR、SOC2 等合规要求落地
5. **安全工具建设** — 搭建 SIEM、WAF、漏洞扫描等安全基础设施

## 标准操作流程 (SOP)
1. 威胁建模：新系统/功能上线前进行威胁分析（STRIDE 模型）
2. 安全测试：代码扫描(SAST) → 依赖检查(SCA) → 动态测试(DAST) → 渗透测试
3. 漏洞管理：发现 → 评级(CVSS) → 分配 → 修复 → 验证 → 关闭
4. 事件响应：检测 → 遏制 → 根除 → 恢复 → 复盘
5. 安全培训：季度安全意识培训 + 开发安全编码培训

## 常见问题处理
- **高危漏洞修复排期困难** → 量化风险（暴露面 × 影响），推动管理层决策
- **开发觉得安全是阻碍** → 安全左移，在 CI 集成自动化扫描，减少人工介入
- **误报率高** → 持续调优规则，建立白名单机制

## 质量标准
- 高危漏洞 72 小时内修复或缓解
- 安全事件 15 分钟内响应
- 安全扫描覆盖率 100%（代码仓库）

## 新人上手指南
- 了解公司安全策略和合规要求
- 熟悉安全工具栈（扫描器、SIEM、WAF 等）
- 阅读近期安全事件复盘报告

## 跨部门协作
- **开发团队** — 漏洞修复、安全编码指导
- **DevOps** — 安全工具集成、基础设施加固
- **法务/合规** — 合规要求落地
- **管理层** — 安全风险汇报
