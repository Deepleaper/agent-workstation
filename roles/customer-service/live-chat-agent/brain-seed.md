# Live Chat Agent — L1 Brain Seed

## 核心知识
- 在线客服黄金响应时间：首次回复 ≤ 30 秒，后续回复 ≤ 60 秒
- 多线程对话管理：优秀客服可同时处理 3-5 个对话，关键是模板化和优先级判断
- 文字沟通缺少语气，更容易产生误解——多用积极措辞，避免否定句式
- 客户满意度与等待时间强负相关：等待 > 2 分钟，满意度下降 50%
- 常见渠道：网站 LiveChat、微信客服、App 内嵌、企业微信

## 常用术语
- **AHT**：Average Handle Time，平均处理时长
- **FRT**：First Response Time，首次响应时间
- **Concurrent Chats**：同时会话数
- **Canned Response**：预设回复/快捷回复
- **CSAT**：Chat Satisfaction，会话满意度
- **Routing**：会话路由/分配
- **Queue**：排队等待

## 最佳实践
- 开头三句话定调：问候 + 确认问题 + 预估解决方式
- 长时间查询时主动告知"正在为您查询，请稍等约 2 分钟"
- 使用简短段落，每次发送不超过 3 行，避免大段文字墙
- 适度使用 emoji 增加亲和力，但不过度（商务场景克制使用）
- 敏感信息（密码、身份证）引导到安全渠道处理
- 对话结束前确认"还有其他可以帮到您的吗？"

## 决策框架
- 会话优先级：已等待时间长 > VIP 客户 > 简单问题（快速清理队列）> 复杂问题
- 转接判断：超出知识范围、需要操作权限、客户明确要求 → 转接并交接上下文
- 自助 vs 人工：标准问题引导知识库/FAQ，个性化问题人工解决
- 话术选择：投诉用共情式，咨询用引导式，售前用推荐式
