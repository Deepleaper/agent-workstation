# 客服工作流程

## 一、接待流程

### 1. 问候阶段
- 3秒内响应用户消息
- 使用标准问候语：「您好，我是XX客服，很高兴为您服务！请问有什么可以帮您？」
- 识别用户情绪，调整沟通语气

### 2. 问题识别
- 倾听用户描述，不打断
- 复述确认：「您的问题是...对吗？」
- 分类归档：售前咨询 / 售后问题 / 投诉建议 / 技术支持

### 3. 问题处理
- 一线问题（FAQ覆盖）：直接解答
- 二线问题（需查询）：告知用户等待，承诺回复时间
- 三线问题（需转接）：说明原因，热转接给专业团队

### 4. 结束确认
- 确认问题是否解决：「请问还有其他可以帮您的吗？」
- 引导满意度评价
- 标准结束语：「感谢您的咨询，祝您生活愉快！」

## 二、投诉处理流程

### 第一步：安抚情绪
- 表达理解：「非常抱歉给您带来不好的体验」
- 不要急于解释或辩解
- 让用户充分表达

### 第二步：了解事实
- 记录关键信息：订单号、时间、具体问题
- 核实系统记录
- 不做任何承诺

### 第三步：提供方案
- 给出1-2个解决方案供选择
- 明确处理时限
- 告知后续跟进方式

### 第四步：执行跟进
- 按承诺时间内完成处理
- 主动回访确认结果
- 记录案例，更新知识库

## 三、升级机制

| 级别 | 触发条件 | 处理人 | 时限 |
|------|---------|--------|------|
| L1 | 常规咨询 | 一线客服 | 即时 |
| L2 | 复杂问题/不满意 | 高级客服 | 2小时 |
| L3 | 重大投诉/媒体曝光 | 客服主管 | 1小时 |
| L4 | 法律纠纷/危机 | 管理层 | 30分钟 |

## 四、常用话术

### 安抚类
- 「非常理解您的心情，换做是我也会着急的」
- 「这个问题确实给您添麻烦了，我马上帮您处理」

### 转接类
- 「这个问题需要专业团队处理，我帮您转接，请稍等」
- 「为了更快解决您的问题，我将您转接给技术专家」

### 拒绝类（委婉）
- 「目前这个功能还在规划中，我帮您记录需求」
- 「根据我们的政策，这种情况暂时无法处理，但我可以帮您...」
